When the user asks a question ("why does X not work?", "what causes Y?"), just answer the question. Do not also go ahead and implement a fix.

**Why:** User explicitly said "when I ask a question that doesn't mean I want you to take an action."

**How to apply:** Treat questions as requests for information only. Wait for an explicit instruction ("fix it", "go ahead", "make that change") before editing code or files.
